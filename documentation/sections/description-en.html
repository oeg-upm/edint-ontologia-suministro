Utility: Description back to ToC

Figure 1 shows the conceptual model of the ontology with its classes and relationships (object properties).

The ontology reuses sensor-related concepts from the (Semantic Sensor Network Ontology) to represent the consumption of utilities by an infrastructure element. An infrastructure element is any facility or urban element that can have utility supply points. These points are features of interest of the measurements made with sensors, which in turn observe an observable property such as electricity consumption, gas consumption, or water consumption.

A meter is a sensor that measures the consumption of a utility. The consumption of a utility reflects the amount consumed over a certain period of time. To represent the consumption, the ontology of quantities, units, dimensions and types is reused, Quantities, Units, Dimensions and DataTypes Ontology. The consumption value is represented by the class QuantityValue which expresses the magnitude and type of a quantity, and is obtained by multiplying a numerical value n by a unit of measurement obtained through the property hasUnit which is related to the class Unit which in turn contains a catalog of units of measurement, for example kilowatt-hour (unit:KiloW-HR) used to measure electricity consumption. The measured period is represented by the class Interval where its start and end times are related to a point in time (Instant). The utility type is a taxonomy that includes concepts such as "active energy", "reactive energy", and "gas". The meter has a meter type which, for example, for energy consumption, can be "external", "processed", or "telemetry". The service contract has a holder and applies to a specific supply point. The contract is also associated with a tariff. A utility meter is a sensor that "measures" a property, which could be, for example, the consumption of electricity, gas, or water.
