En la Figura 1 se muestra el modelo conceptual de la ontología con sus clases y relaciones (propiedades de objeto).

La ontología reutiliza conceptos relacionados a sensores de la (Semantic Sensor Network Ontology) para representar el consumo de suministros de un elemento de la infraestructura. Un elemento de la infraestructura es cualquier equipamiento o elemento urbano que puede tener puntos de suministro de servicios. Estos puntos son características de interés de las mediciones hechas con sensores, que a su vez observan una propiedad observable tal como el consumo de electricidad, el consumo de gas o el consumo de agua.

Un medidor es un sensor que mide el consumo del suministro. Los consumos de un suministro reflejan la cantidad consumida en cierto período de tiempo. Para representar los consumos se reutiliza la ontología de cantidades, unidades, dimensiones y tipos, Quantities, Units, Dimensions and DataTypes Ontology. El valor del consumo se representa mediante la clase QuantityValue que expresa la magnitud y el tipo de una cantidad y se obtiene multiplicando un valor numérico n por una unidad de medida que se obtiene a través de la propiedad hasUnit que se relaciona con la clase Unit que contiene un catálogo de uniddes de medida, por ejempo kilovatio-hora (unit:KiloW-HR) utilizado para medir el consumo de electricidad. El período medido se representa mediante la clase Interval en donde su hora de inicio y hora de fin se relacionan con un instante de tiempo (Instant). El tipo de suministro es una taxonomía en donde se tienen conceptos como "energía activa", "energía reactiva" y "gas". El medidor tiene un tipo de medidor que por ejemplo para consumo de energía puede ser "externo", "procesado" o "telemedida". El contrato de servicio tiene un titular y se aplica un determinado punto de suministro. El contrato también se asocia con una tarifa. Un medidor de suministro es un sensor que "observa" una propiedad que puede ser por ejemplo el consumo de electricidad, gas o agua.
